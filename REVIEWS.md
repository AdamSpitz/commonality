#### Bug Found â€” âš ď¸Ź `decodeContractMetadataUpdatedEvent` uses wrong field name

In [sdk/src/utils/eventDecoder.ts:249](sdk/src/utils/eventDecoder.ts#L249):
```ts
metadata: args.uri as string,  // BUG: field is named 'metadata' not 'uri'
```
The `ContractMetadataUpdated` ABI event field is named `metadata`, not `uri`. When events come from the event cache, `args.uri` is `undefined`, so `metadata` is always `undefined`. The fold (`foldProject`) does `event.uri || event.metadata` which also resolves to `undefined`. Result: pubstarter project metadata (IPFS CID) is never populated when using the event cache path.

Tests pass because fold tests construct events directly with `uri: METADATA_CID`, bypassing the decoder. **This bug is not caught by any existing test.**

Fix: Change `args.uri` â†’ `args.metadata` in the decoder.
