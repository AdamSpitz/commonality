# Assurance contracts

You want to fund a project, but only if enough other people fund it too. You don't want to put in $50 if nobody else shows up.

An assurance contract solves this: you pledge your $50 toward a goal (say, $1,500 for a neighborhood block party). If the goal is met by the deadline, the money goes to the project. If not, you get refunded. You risk nothing.

This isn't a new idea — Kickstarter works this way. The point is that assurance contracts are a surprisingly powerful mechanism, and we should be using them for a lot more things than consumer products. Community projects, public goods, content funding, even political organizing — anywhere the problem is "everyone would chip in if they knew everyone else would too," an assurance contract eliminates the coordination barrier.

## How this shows up in practice

A project creator sets up an assurance contract with a funding goal and a deadline. Donors pledge. The pledge page shows how much has been pledged and how much is needed — that visible progress is itself a coordination signal, because it tells potential pledgers "this is close, your contribution could push it over the line."

If the goal is met, the project is funded. If not, all pledges are refunded automatically. No chasing refunds, no trust required.

Assurance contracts can also be recurring (monthly pledges toward an ongoing goal) and can have standby conditions ("activate only if X happens") — which is the foundation for credible threats.
