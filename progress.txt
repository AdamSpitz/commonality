**Root cause analysis:**
The E2E test failure is due to a fundamental architectural mismatch between:
1. The SDK (`@commonality/sdk`) - designed for test environments with direct private key access
2. The UI - using wagmi which abstracts away private keys and uses browser wallets

**The problem in detail:**

The SDK's actions (like `believeStatement`, `createAndSignStatement`) expect a `TestClients` object with:
```typescript
{
  walletClient: WalletClient,  // Must have .account property for signing
  publicClient: PublicClient,
  account: Address
}
```

In test/integration environments, the wallet client is created with `createWalletClient({ account: privateKeyToAccount(privateKey), ... })`, which gives it a full viem account object with signing capabilities.

In the UI using wagmi:
- `useWalletClient()` returns a wallet client from the connected browser wallet
- wagmi's mock connector (used in E2E tests) doesn't support the same signing methods
- Error progression:
  1. First: "Could not find an Account" - walletClient.account was undefined
  2. After adding `{ address }`: "Account type undefined is not supported" - account missing `type` property
  3. After adding `{ address, type: 'json-rpc' }`: "method eth_sendTransaction not supported" - json-rpc accounts expect RPC-level signing, not available in mock connector

**Why this is a larger task:**

Fixing this properly requires ONE of these approaches:

1. **Refactor the SDK** to not require `walletClient.account` for signing
   - Make SDK actions use wagmi's hooks directly (useWriteContract, etc.)
   - This is a significant SDK refactoring affecting all actions

2. **Create a UI-specific wrapper layer** around SDK actions
   - Wrapper translates between wagmi patterns and SDK expectations
   - Maintains SDK as-is for integration tests
   - Requires designing and implementing the wrapper API

3. **Make E2E tests use real wallet simulation** with private keys
   - Use wagmi's `privateKeyToAccount` to create proper accounts in E2E tests
   - This might require changes to how wagmi's mock connector works
   - May need to inject private key into browser context (security implications)

**Recommendation for breaking down the task:**

The smallest viable approach is #3 - modify E2E test wallet setup:
