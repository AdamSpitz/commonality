# No prerequisites

Collective action normally fails because it has prerequisites: you need everyone to agree first, you need critical mass first, you need to be sure it'll work first, you need permission first, you need to commit to a big change first. Commonality is designed so that none of these apply. Every action stands on its own — nothing needs to happen before you can act, and nothing you do depends on anything else succeeding.

Each of these is a specific precondition that the system eliminates:

  - [Individualization.](./individualization.md) Removes "we all need to agree first." Every action is a personal decision that doesn't require anyone else's buy-in.
  - [Scales down.](./scales-down.md) Removes "we need critical mass first." The system is useful even for a single project or a single user.
  - [Nearly costless to try-and-fail.](./costless-to-try.md) Removes "I need to be sure it'll work first." Assurance contract refunds, revocable delegation, free statement signing.
  - ["Publish, then filter."](./publish-then-filter.md) Removes "someone needs to approve this first." No gatekeepers. Anyone can publish whatever they want.
  - [Dial, not switch.](./dial-not-switch.md) Removes "I need to commit to a big change first." Every adoption path starts at a level identical to what you're already doing.

Of course a cause still does need to actually gather users — get donors to pledge money, get doers to start projects. But the [pitches](./pitches.md) to each user are straightforward precisely because there are no prerequisites: you don't need to have faith that the entire enterprise will take over the world.
