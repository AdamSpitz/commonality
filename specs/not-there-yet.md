
  Project & Funding Features

  - Application lifecycle for projects: From vague cause → measurable goals
  → running applications
  - Multiple entry/exit points: Projects can start from causes, or causes
  can emerge from existing projects
  - Nano-exchanges: Trading tokens between different causes

  Philosophical/Aspirational

  - Optimism in humanity: Belief that people are more sane/good than current
   systems suggest
  - Decentralizing control while maintaining efficiency: Getting "efficiency
   of centralized decision-making but flexibility and transparency of having
   control yourself"

  Development Process

  - AI-assisted ontology generation: Using AI to propose knowledge graph
  ontologies
  - Prototype-first approach: Building working prototype before strategic
  outreach
  - Generator-based testing: Generating billions of simulated users/causes
  to test scalability
