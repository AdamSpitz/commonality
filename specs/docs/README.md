# Documentation

## Roles

Different roles require different kinds of documentation:

  - [Founder](./founder-docs.md): The whole big picture. What do we want this project to accomplish in the world? Why do we think it could accomplish those goals? Also high-level specs for what we're building.
  - [Product manager](./product-manager-docs.md): Turning the overall vision into a high-level product spec.
  - [Technical lead](./tech-lead-docs.md): Turning the high-level specs into medium-level specs. Involves making technical decisions, etc.
  - [Dev](./dev-docs.md): Turning the medium-level specs into running code.
  - [User](./user-docs.md) (or LLM whose role is to assist the user).
