# Content

Let's try to think through "where does content come from?"

## Kinds of content

## Statements

Let's start with statements. This should be easy because it doesn't require any money as an input, and not much effort either, and it's very amenable to AI assistance.

When a new user begins using the system, maybe the UI or AI assistant shows him various possible choices for areas that he might be interested in (politics, music, sports, etc.).

A thought: I wonder whether "I am interested in X" might itself be useful as a statement within the system. That is, conservatives and progressives are going to be hitting the Like button on very different political statements, but any of those statements is also going to imply "I am interested in politics". I wonder whether those "I am interested in X" statements might serve as a list of suggested areas of interest that this system can be used for.

In fact, the UI or AI assistant could *start* by showing the user a list of those:
  - I am interested in politics.
  - I am interested in music.
  - etc.

That's interesting. Maybe it's useful to think of how-to-flesh-out-the-statement-space as a combination of "bottom-up" and "top-down":
  - Bottom-up: Creating/discovering implications of already-signed statements. X -> Y and you already believe X so you probably believe Y. "I am a conservative" -> "I am interested in politics." There isn't necessarily any need to get the user to explicitly sign Y.
  - Top-down: Fleshing-out / pinning down what the user believes. X1 -> Y, X2 -> Y and the user believes Y; the UI / AI assistant can *suggest* that maybe the user might believe X1 or X2. ("I am interested in politics" -> do you believe "I am a conservative" or "I am a progressive"?)

(I'm not sure "bottom-up" and "top-down" are the right terms; that's just what came to mind when I was imagining this as going from conservative -> politics vs going from politics -> conservative or progressive.)

In practice, I expect that the fleshing-out process (for assisting any particular user) is going to be a mix, fluidly going back and forth between bottom-up and top-down; it's not like it needs to be one or the other.

e.g.
  - "I'm new here. What the heck is this system for?"
  - "This is a system for funding public goods aligned with causes, (blah blah explain the system briefly). To get started, let's find out what you're interested in and what you believe. Here are some options: politics, music, sports, crypto, etc." (In another panel, the UI shows various statements that the user can click Like on: "I'm interested in politics", etc. Importantly, the UI that displays the statement is *not* generated by the LLM: the LLM tells the UI to show statementId1, statementId2, etc., but then the actual statements and the Like button and so on are done by straightforward non-AI code.)
  - User clicks the checkbox on the "I'm interested in politics" and "I'm interested in crypto" statements. The LLM notices this because it's somehow subscribed/polling-for those kinds of events from this user, via the indexer. We may have to invent some way for the indexer to convey this information; I'm not sure what's already implemented.
  - "Okay, let's tackle politics first. Do you lean left or right?" Two statements appear in the other panel.
  - User checks "I generally tend to lean right-wing."
  - "Are you more of a libertarian or a conservative?" Two statements appear in the other panel.
  - User types to the LLM: "Not really either? Or, like, some of each? I'm sympathetic to both. I'm not especially religious, I don't see anything wrong with being gay, I don't mind abortion, but I do think a bunch of the conservative ideas about tradition make a lot of sense."
  - LLM composes a new statement that appears in the other panel, a more-polished version of what the user said.
  - User clicks the checkbox on that statement.
  - "Okay, so just to make sure of the specifics: you're probably also in favour of harsher sentences for repeat offenders; generally supportive of markets; against DEI programs; etc. Right?" A bunch of statements appear in the other panel. These ones show up flagged as "you haven't explicitly signed these, but based on your other beliefs the system thinks you probably believe them."
  - User checks the box on some or all of them.
  - LLM: "Okay, now let's move on to crypto. Are you a fan of Bitcoin? Ethereum? Solana?" More statements appear.
  - etc.

Okay, this is something that the existing conceptspace UI doesn't handle. We're gonna need a new UI page for it. See [conceptspace explorer](/specs/subsystems/conceptspace/explorer.md).

### No need for a flowchart

This doesn't need a big static diagram with a bunch of if-then flows like "the user just signed S1, now ask about S2 and S3". A general-purpose LLM can probably do a decent job of generating those kinds of things on the fly.
