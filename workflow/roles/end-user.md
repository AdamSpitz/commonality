# End user documentation

Which documentation you'll read will depend on which of the top-level [UI domains](/specs/product/ui-domains.md) you land on; each has a landing page.

See [ui/src/domains](/ui/src/domains/) to find the code for the landing pages.

But honestly, the right thing to do, if you're acting as an end user, is probably to read the actual website. You can use `scripts/services.sh` to start up a local deploy if one isn't already running, then find the stable local top-level UI URLs at `localhost:8088/`.
