Different roles need different kinds of documentation. This file is the guide to what to read (and what *not* to bother reading).

  - [Founder (vision/strategy)](./founder.md)
  - [Product manager (what to build)](./product-manager.md)
  - [Tech lead (how to build it)](./tech-lead.md)
  - [Dev (implementation)](./developer.md)
  - [End-user documentation *writer*](../../specs/user-docs.md)
  - [End user](./end-user.md)

## Testing/validation roles

If your role is to test or validate some aspect of the project, it probably makes sense to read the documentation meant for some of the above roles - just keeping in mind what your role is.

e.g.
  - "make sure this functionality works" -> read the developer docs to figure out where the functionality lives and how you might want to test it
  - "make sure the landing page for this site is compelling" -> read the founder-level docs to figure out what the purpose of the site is
  - etc.
